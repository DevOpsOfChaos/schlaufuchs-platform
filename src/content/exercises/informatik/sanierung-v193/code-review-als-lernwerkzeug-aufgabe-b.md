## Aufgabe

Übertrage das Thema Code-Review als Lernwerkzeug auf ein zweites Beispiel und prüfe, wo ein Fehlerfall entstehen kann.

Arbeite in kurzen Abschnitten:

1. **Ausgangslage:** Beschreibe die Situation in zwei bis drei Sätzen.
2. **Erwartung:** Notiere, welches Verhalten, welche Datenstruktur oder welcher Zustand korrekt wäre.
3. **Prüfung:** Lege einen kleinen Test, eine Kontrolle oder ein Minimalbeispiel fest.
4. **Bewertung:** Erkläre, welche Entscheidung du daraus ableitest.

## Zusatzfrage

Welche Information müsste fehlen, damit deine Entscheidung unsicher wird?
