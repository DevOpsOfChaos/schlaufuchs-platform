## Arbeitsauftrag

Bearbeite die Aufgaben zunächst konzeptionell. Es geht nicht nur darum, Syntax aufzuschreiben, sondern auch darum, sinnvolle Funktionsgrenzen zu erkennen.

## Aufgabe 1

Gegeben ist die Idee einer Funktion, die zwei Zahlen addiert.

1. Formuliere einen passenden Funktionskopf in C.
2. Nenne die Parameter.
3. Nenne den Rückgabetyp.

## Aufgabe 2

Eine Funktion soll prüfen, ob eine Zahl gerade ist.

1. Welcher Rückgabetyp ist sinnvoll?
2. Welcher Parameter wird benötigt?
3. Formuliere einen möglichen Funktionsnamen.

## Aufgabe 3

Ordne zu, ob eher `void` oder ein Rückgabewert sinnvoll ist.

1. Eine Funktion gibt einen Begrüßungstext aus.
2. Eine Funktion berechnet den Flächeninhalt eines Rechtecks.
3. Eine Funktion prüft, ob ein Passwort lang genug ist.

## Aufgabe 4

Ein Programm enthält mehrfach denselben Code, um die Summe zweier Zahlen zu bilden und auszugeben.

1. Erkläre kurz, warum hier eine Funktion sinnvoll ist.
2. Entscheide, ob Rechnen und Ausgeben besser in einer oder in zwei Funktionen getrennt werden sollten.
3. Begründe deine Entscheidung kurz.

## Aufgabe 5

Schreibe eine kleine Funktion in C, die die größere von zwei ganzen Zahlen zurückgibt.

Die Funktion soll:

- zwei Parameter haben,
- einen `int` zurückgeben,
- eine `if`-Bedingung enthalten.

## Hinweise

- Ein Funktionskopf besteht aus Rückgabetyp, Name und Parameterliste.
- `void` eignet sich für Funktionen ohne Rückgabewert.
- Gute Funktionsnamen beschreiben die Aufgabe klar.
- Eine Funktion sollte möglichst eine überschaubare Aufgabe übernehmen.

## Selbstkontrolle

Diese Punkte sollten am Ende klar sein:

- Nicht jede Funktion braucht einen Rückgabewert.
- Parameter machen Funktionen wiederverwendbar.
- In Aufgabe 5 ist eine Vergleichslogik nötig.
