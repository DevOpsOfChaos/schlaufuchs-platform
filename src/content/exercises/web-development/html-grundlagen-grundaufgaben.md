Diese Aufgabe hält die Grundidee von HTML bewusst einfach: Du sollst verstehen, **warum HTML Inhalte strukturiert und nicht direkt gestaltet**.

## Lernziel

Nach dieser Aufgabe solltest du sicher einordnen können,

- welche Rolle HTML im Web Development hat,
- warum HTML Struktur statt Gestaltung beschreibt,
- weshalb diese Unterscheidung für Webseiten wichtig ist.

## Aufgabe 1 – Struktur oder Gestaltung?

Ordne mehrere Aussagen danach, ob sie eher zu

- HTML,
- CSS,
- oder allgemeiner Webgestaltung

gehören.

Begründe kurz.

## Aufgabe 2 – Struktur fachlich erklären

Erkläre in eigenen Worten,

- warum Überschriften, Absätze und Listen zur HTML-Struktur gehören,
- warum HTML nicht einfach nur „das Aussehen der Seite“ ist,
- weshalb diese Trennung für gute Webentwicklung wichtig ist.

## Aufgabe 3 – HTML im Gesamtbild einordnen

Beschreibe, welche Rolle HTML im Zusammenspiel mit CSS hat.

Nutze dabei Begriffe wie Struktur, Inhalt, Darstellung oder Gestaltung.

## Aufgabe 4 – Aussagen prüfen

Ordne als **richtig** oder **falsch** ein:

- HTML beschreibt die Struktur von Inhalten.
- HTML ist vor allem für Farben und Layout zuständig.
- HTML bildet eine Grundlage jeder Webseite.
- Gute Webentwicklung trennt Struktur und Gestaltung.

Begründe kurz.

## Hinweise

- Denke HTML zuerst als Struktur- und Bedeutungsbeschreibung.
- Trenne inhaltliche Gliederung bewusst von optischer Gestaltung.
- Gute Grundlagen machen spätere Webthemen deutlich leichter.

## Selbstkontrolle

Du bist auf einem guten Stand, wenn du am Ende sicher sagen kannst,

- was HTML grundsätzlich leistet,
- warum HTML nicht mit CSS verwechselt werden darf,
- weshalb Struktur für Webseiten so zentral ist.
