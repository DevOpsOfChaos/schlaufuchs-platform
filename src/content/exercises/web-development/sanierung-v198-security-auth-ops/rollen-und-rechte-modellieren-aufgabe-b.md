## Aufgabe

Formuliere einen kurzen Prüfplan für **Rollen und Rechte modellieren**. Der Plan soll vor einem Release helfen, einen typischen Fehler nicht erst in Produktion zu entdecken.

Baue den Plan so auf:

1. eine manuelle Sichtprüfung,
2. eine technische Prüfung,
3. eine passende Fehlermeldung oder Log-Aussage,
4. ein Kriterium, wann der Punkt als erledigt gilt.

## Erwartung

Schreibe keine lange technische Abhandlung. Wichtig ist eine nachvollziehbare Reihenfolge: Beobachten, abgrenzen, prüfen, verbessern.
