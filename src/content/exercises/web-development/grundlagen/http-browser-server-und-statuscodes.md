## Arbeitsauftrag

Bearbeite die Aufgaben so, dass immer klar bleibt, welche Rolle der Browser und welche Rolle der Server hat.

## Aufgabe 1

Erkläre in eigenen Worten:

1. Was der Browser macht.
2. Was der Server macht.
3. Was HTTP regelt.

## Aufgabe 2

Ordne zu:

- Anfrage des Browsers
- Antwort des Servers
- erfolgreiche Antwort
- Ressource nicht gefunden
- interner Serverfehler

Begriffe:

- Request
- Response
- 200
- 404
- 500

## Aufgabe 3

Erkläre kurz:

1. Warum eine Webseite oft aus mehreren HTTP-Anfragen besteht.
2. Warum ein Bild auf einer Seite nicht automatisch „im HTML drin“ sein muss.

## Aufgabe 4

Beschreibe einen einfachen Ablauf:

- Eine Person öffnet eine Webadresse.
- Die HTML-Seite wird geladen.
- Danach werden noch CSS und ein Bild geladen.

Schreibe diesen Ablauf in 3 bis 5 klaren Schritten auf.

## Aufgabe 5

Erkläre kurz:

1. Warum Statuscodes bei der Fehlersuche helfen.
2. Warum `404` etwas anderes bedeutet als `500`.

## Hinweise

- Der Browser fordert Inhalte an.
- Der Server liefert Antworten.
- Eine sichtbare Seite entsteht oft aus mehreren geladenen Ressourcen.
