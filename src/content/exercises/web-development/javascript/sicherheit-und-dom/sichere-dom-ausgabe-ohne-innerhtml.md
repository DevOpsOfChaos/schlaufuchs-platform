## Aufgabe

Du bekommst eine kleine Web-Oberfläche, die grundsätzlich funktioniert, aber an einer Stelle unklar, fragil oder schwer wartbar ist.

Bearbeite die Situation passend zum Thema **Sichere DOM-Ausgabe ohne innerHTML**.

## Szenario

Eine Lernplattform zeigt eine neue Funktion. Im Idealfall wirkt sie okay, aber bei einem Randfall entsteht ein Problem:

- eine Eingabe bleibt unklar,
- ein Zustand wird nicht erklärt,
- ein Fehler überschreibt eine gute Nutzerführung,
- oder der Code ist so gemischt, dass spätere Änderungen riskant werden.

## Deine Schritte

1. Beschreibe den problematischen Zustand in Alltagssprache.
2. Benenne die technische oder strukturelle Ursache.
3. Schlage eine konkrete Verbesserung vor.
4. Formuliere eine kurze Prüffrage, mit der ein Review den Fehler später wiederfinden würde.

## Erwartung

Deine Antwort soll nicht nur sagen, „man müsste es besser machen“. Sie soll zeigen, **was genau** besser wird und **woran** man das prüfen kann.
