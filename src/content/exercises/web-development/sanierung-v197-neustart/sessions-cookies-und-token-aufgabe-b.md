## Aufgabe

Entwirf eine robuste Fehler- und Kontrollstrategie für denselben Themenbereich.

Arbeite mit einer kurzen Skizze oder Tabelle. Deine Lösung soll nicht aus Code bestehen, sondern aus einer fachlich sauberen Einordnung.

## Ausgangssituation

Eine Webseite sendet Daten an eine API. Die Antwort entscheidet, ob die Oberfläche Erfolg, Warnung oder Fehler anzeigt.

## Bearbeitungsschritte

1. Benenne die beteiligten Teile: Browser, API, Serverlogik und Datenquelle.
2. Beschreibe den normalen Ablauf in vier bis fünf Schritten.
3. Ergänze mindestens einen Fehlerfall.
4. Erkläre, welche Prüfung nicht allein im Frontend bleiben darf.
5. Formuliere eine kleine Testidee.

## Erwartung

Am Ende soll klar sein, welche Aufgabe welcher Teil der Webanwendung übernimmt.
