## Aufgabe

Bearbeite das Szenario zu **Optimistic UI mit Rollback planen**.

## Szenario

Ein Like wird sofort angezeigt, bevor der Server bestätigt.

## Arbeitsauftrag

1. Ordne die Situation dem genauen Unterthema zu.
2. Erkläre die naheliegende Verwechslung: Optimistische Anzeige wird ohne Fehlerweg gebaut.
3. Nutze diesen Prüfschritt: Plane, wie die UI bei Serverfehler zurückgesetzt wird.
4. Formuliere eine Gegenprobe, bei der sich ein Wert, Zustand oder Kontext ändert.

## Erwartete Abgabe

- eine klare Einordnung,
- eine fachliche Abgrenzung,
- ein konkreter Prüfschritt,
- eine kurze Gegenprobe.
