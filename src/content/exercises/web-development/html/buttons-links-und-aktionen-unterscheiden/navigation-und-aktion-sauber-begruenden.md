## Aufgabe 1: Zuordnen

Ordne die folgenden Situationen jeweils als Link oder Button ein und begründe kurz:

- „Zum Impressum“
- „Suche öffnen“
- „Formular absenden“
- „Mehr zum Thema lesen“

## Aufgabe 2: Denkfehler prüfen

Jemand sagt: „Wenn etwas wie ein Button aussieht, muss es auch ein Button sein.“

Erkläre, warum diese Aussage für HTML zu kurz greift.

## Aufgabe 3: Oberfläche deuten

Eine große blaue Fläche mit „Konto erstellen“ führt auf eine neue Registrierungsseite.

Begründe, warum dafür ein Link sinnvoller sein kann als ein Button – auch wenn beide ähnlich gestaltet werden könnten.
