# Paketquellen und Repositories

Paketquellen sind definierte Orte, aus denen die Paketverwaltung Software und Aktualisierungen bezieht.

Distributionen liefern Software meist über kuratierte Repositories aus.

## Einordnung

Der Begriff ist besonders hilfreich, wenn einzelne Beobachtungen nicht isoliert, sondern als Teil eines größeren Zusammenhangs verstanden werden sollen. In Lern- und Praxiszusammenhängen dient er als Orientierung, bevor Spezialfälle oder Rechenverfahren betrachtet werden.

## Zentrale Aspekte

Repository-Listen bestimmen, welche Pakete gefunden werden.

Signaturen und Schlüssel helfen, Herkunft und Unverändertheit zu prüfen.

Fremdquellen können neuere Software liefern, erhöhen aber Wartungs- und Vertrauensfragen.

## Abgrenzung

Ein Repository ist nicht nur ein Download-Ordner. Es enthält Metadaten, Versionen und Vertrauensinformationen.

## Beispiele und typische Situationen

Eine Distribution stellt Sicherheitsupdates über offizielle Quellen bereit.

Ein Hersteller kann eine eigene Paketquelle für spezielle Software anbieten.

## Häufige Missverständnisse

Häufig werden beliebige Paketquellen hinzugefügt, um eine neue Version zu bekommen. Das kann spätere Updates komplizieren.

## Kurz zusammengefasst

Paketquellen bestimmen, welcher Software das System vertraut.
