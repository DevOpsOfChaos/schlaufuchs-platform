Layout beschreibt, wie Inhalte auf einer Webseite räumlich angeordnet werden. Responsive Design bedeutet, dass diese Anordnung auf unterschiedliche Bildschirmgrößen, Eingabegeräte und Inhaltsmengen reagieren kann.

## Einordnung

Webseiten werden auf Smartphones, Tablets, Laptops und großen Monitoren genutzt. Ein gutes Layout ist deshalb kein starres Bild, sondern ein System aus Regeln, Abständen und Prioritäten.

## Warum das Thema wichtig ist

## Zentrale Bestandteile

- **Inhaltsfluss:** Die natürliche Reihenfolge und Ausbreitung von Elementen.
- **Flexbox:** Ein Layoutsystem für eindimensionale Verteilung.
- **Grid:** Ein Layoutsystem für zweidimensionale Raster.
- **Breakpoint:** Ein Punkt, an dem Layoutregeln geändert werden.

## Grundprinzip

Das Thema wird zuerst über Begriffe, Zweck und typische Entscheidungen verstanden. Erst danach lohnt sich der Blick auf Spezialfälle, Codebeispiele, Messwerte oder Rechenschritte.

## Abgrenzung zu Detailseiten

Detailseiten erklären konkrete Methoden, Situationen oder Fehlerbilder. Diese Überblicksseite ist allgemeiner: Sie beschreibt, worum es im Themenfeld geht, warum es gebraucht wird und wie die folgenden Lernseiten eingeordnet werden können.

## Beispielhafte Anwendung

Eine Kartenliste kann auf großen Bildschirmen mehrspaltig stehen und auf kleinen Bildschirmen einspaltig werden. Lesbarkeit und Bedienbarkeit bleiben entscheidend.

## Zusammenfassung

Eine gute Überblicksseite ersetzt keine Detailseite. Sie ordnet ein Thema so ein, dass einzelne Spezialfälle später verständlich werden. Wer zuerst Begriffe, Zweck und Grenzen kennt, kann Aufgaben und Beispiele deutlich ruhiger lesen.
