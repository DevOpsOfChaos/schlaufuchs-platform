# Web-Animationen

Web-Animationen verändern Eigenschaften über Zeit. Sie können Orientierung geben, Zustände erklären oder rein dekorativ wirken.

## Einordnung

Animation ist ein Gestaltungsmittel, aber auch ein Performance- und Accessibility-Thema. Gute Animationen unterstützen Verständnis, schlechte lenken ab oder machen eine Seite schwer nutzbar.

## Zentrale Aspekte

- Animationen sollten eine erkennbare Funktion haben.
- Kurze Übergänge können Zustandswechsel verständlicher machen.
- Nutzer mit Bewegungsreduktion müssen berücksichtigt werden.
- Performance hängt stark von animierten Eigenschaften ab.

## Beispiele und Zusammenhang

Ein Menü kann beim Öffnen leicht einblenden. Eine dauernde große Hintergrundanimation kann dagegen Aufmerksamkeit und Leistung kosten.

## Abgrenzung

Animation ersetzt keine klare Struktur. Ein Nutzer muss Zustand und Bedienung auch ohne Bewegung verstehen.

## Häufige Missverständnisse

Bewegung als reine Dekoration einzusetzen und `prefers-reduced-motion` zu ignorieren.

## Kurz zusammengefasst

- Animationen sollten Orientierung geben.
- Reduzierte Bewegung ist ein wichtiger Nutzerwunsch.
- Performancefreundliche Eigenschaften vermeiden Ruckeln.
