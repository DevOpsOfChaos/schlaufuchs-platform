# Progressive Enhancement

![Schaubild zu Progressive Enhancement.](/schlaufuchs-platform/images/overviews/v149/progressive-enhancement.svg)

*Das Schaubild ordnet Basisfunktion, Verbesserung und Zusatzverhalten ein.*



## Begriff und Zweck

Progressive Enhancement bedeutet, eine Webseite zuerst auf einer stabilen Grundfunktion aufzubauen und zusätzliche Fähigkeiten schrittweise zu ergänzen. Inhalt, Struktur und grundlegende Bedienung sollen auch dann funktionieren, wenn JavaScript langsam lädt, blockiert ist oder einzelne APIs nicht verfügbar sind.

## Wichtige Zusammenhänge

Wichtige Teilaspekte sind:

- semantisches HTML als Basis
- CSS als Verbesserung der Darstellung
- JavaScript als zusätzliche Interaktion
- Fehler- und Ladezustände als Teil des Designs

Diese Punkte gehören zusammen. Ein einzelner Begriff erklärt meist noch nicht, ob eine Lösung zuverlässig, verständlich oder fachlich passend ist. Deshalb lohnt sich die Einordnung: Was wird betrachtet, welche Annahmen gelten und welche Grenzen hat das Modell oder Werkzeug?

## Beispiele und typische Verwendung

Ein Formular kann grundsätzlich per HTML absendbar sein und durch JavaScript komfortabler validiert werden. Eine Navigation kann als echte Linkliste funktionieren und später mit Animation oder Zustand erweitert werden. So bleibt die Seite robuster gegenüber Netzwerkproblemen und unterschiedlichen Geräten.

## Abgrenzung

Progressive Enhancement ist nicht gleichbedeutend mit veraltetem Webdesign. Es ist ein Qualitätsprinzip, das moderne Oberflächen widerstandsfähiger macht. Es grenzt sich von Ansätzen ab, bei denen ohne JavaScript nur eine leere Seite sichtbar ist.

## Häufige Missverständnisse

Ein häufiger Irrtum ist, dass alle Nutzenden immer die gleiche moderne Umgebung haben. In der Praxis unterscheiden sich Browser, Geräte, Verbindungen, Rechte und Hilfstechniken. Progressive Enhancement reduziert die Abhängigkeit von perfekten Bedingungen.
