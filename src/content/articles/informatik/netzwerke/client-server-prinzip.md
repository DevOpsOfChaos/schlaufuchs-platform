## Warum dieses Thema wichtig ist

Viele digitale Dienste wirken im Alltag selbstverständlich. Technisch steckt dahinter aber oft dieselbe Grundidee: Ein Teil fordert etwas an, ein anderer Teil liefert etwas zurück.

## Die Grundidee

Ein **Client** ist der Teil, der eine Anfrage stellt. Ein **Server** ist der Teil, der auf diese Anfrage reagiert und einen Dienst bereitstellt.

Typisch ist also dieser Ablauf:

1. Der Client schickt eine Anfrage.
2. Der Server verarbeitet sie.
3. Der Server sendet eine Antwort zurück.

## Rollen statt feste Geräte

Client und Server sollten zuerst als **Rollen** im Ablauf verstanden werden. Darum ist die wichtigere Frage nicht „Welches Gerät ist das?“, sondern „Wer fordert an und wer stellt bereit?“.

## Was der Server bereitstellt

Ein Server stellt oft nicht nur Daten, sondern einen **Dienst** bereit, zum Beispiel eine Website, eine Anmeldung, eine Datenbankabfrage oder eine Schnittstelle für eine App.

## Typische Alltagsbeispiele

- Ein Browser lädt eine Website.
- Eine Wetter-App ruft aktuelle Daten ab.
- Ein Messenger sendet Nachrichten an einen Dienst.
- Ein Schulportal prüft eine Anmeldung und zeigt Daten an.
