## Musterlösung

### Aufgabe 1

Eine gute Antwort nennt nicht nur Wörter, sondern Rollen. Bei **API-Verträge und Datenfluss** sollten die zentralen Begriffe so erklärt werden, dass ihr Zusammenhang sichtbar wird. Wichtig ist besonders: Eine API beschreibt, wie Systeme miteinander sprechen.

### Aufgabe 2

Eine passende Situation beschreibt zuerst den Kontext und erst danach die konkrete Handlung. Sinnvoll ist zum Beispiel diese Struktur:

- Welche Daten, Eingaben oder Zustände liegen vor?
- Welche Regel, Auswahl oder Modellierung wird verwendet?
- Welches Ergebnis entsteht und welche Grenze bleibt sichtbar?

Damit wird das Thema nicht als isolierter Einzelfall behandelt.

### Aufgabe 3

Die Aussage ist zu kurz, weil sie einen wichtigen Teilaspekt ausblendet. Präziser ist:

> Der Vertrag umfasst Datenformat, Pflichtfelder, Antwort und Fehlerfälle.

Zusätzlich sollte beachtet werden: Eine API ist nicht nur eine URL, sondern eine fachliche Vereinbarung.

### Aufgabe 4

Beim Transfer bleibt die Grundidee erhalten, aber die konkrete Situation ändert sich. Eine gute Lösung zeigt deshalb, welche Struktur gleich bleibt und welche Begriffe im neuen Beispiel anders besetzt sind.

## Ergebnis

Die Aufgabe ist gut gelöst, wenn Begriff, Beispiel, Grenze und Begründung zusammenpassen. Eine reine Stichwortliste reicht nicht aus.
