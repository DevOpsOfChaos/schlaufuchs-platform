## Aufgabe 1

Weil HTTP zustandslos ist. Jeder neue Request sieht für den Server zunächst wie ein neuer Einzelzugriff aus, wenn keine zusätzliche Wiedererkennungslogik genutzt wird.

## Aufgabe 2

Beim Cookie werden Informationen oder Kennungen im Browser mitgeführt. Bei der Session liegen die eigentlichen Zustandsdaten auf dem Server, während der Browser meist nur eine Kennung mitliefert.
