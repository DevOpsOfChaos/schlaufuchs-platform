# Plattform – konkrete Fach-Landingpage `web-development`

## Zweck dieser Datei
Diese Datei beschreibt die **konkrete repo-nahe Zielstruktur** für die Fachbereichs-Landingpage `web-development`.

Sie baut auf drei Ebenen auf:
- bestehender Repo-Iststand mit Fachnavigation und Collections
- bereits geplante HTML/CSS-Fachstruktur in `docs/`
- geplante Aufgabenintegration über `exercises`

## Warum `web-development` und nicht einfach `HTML/CSS`?
Im Repo ist der Fachbereich bereits als `web-development` angelegt. Die bisherige inhaltliche Planung dreht sich zwar stark um HTML/CSS, aber diese Inhalte sollen **innerhalb** dieses Fachbereichs organisiert werden – nicht als eigener konkurrierender Hauptbereich. 

## Repo-Anschluss
### Subject-Slug
- `web-development`

### Andockpunkte
- `subjects`
- `articles`
- `exercises`

## Rolle der Fachseite
Die Web-Development-Fachseite soll drei Einstiegsarten unterstützen:
- Lernende, die Web-Grundlagen erst verstehen wollen
- Lernende, die gezielt HTML/CSS-Themen suchen
- Lernende, die mit kleinen Praxis- und Code-Aufgaben üben möchten

## Ziel der Fachseite
Die Seite soll:
1. Web Development als Fach klar eröffnen
2. HTML und CSS als Einstiegsachse sichtbar machen
3. Wissensseiten und Übungen gemeinsam präsentieren
4. Grundlagen und spätere Praxis sauber trennen
5. Code-/Preview- und Aufgabenlogik sichtbar machen

## Empfohlener Aufbau
## 1) Hero-Bereich
### Inhalt
- Titel: `Web Development`
- kurze Fachbeschreibung
- ein Satz zum Nutzen des Bereichs
- 2 primäre Einstiege

### Beispiel-CTA
- `Mit HTML starten`
- `Direkt Aufgaben üben`

### Kernaussage im Hero
Web Development auf Schlaufuchs verbindet:
- Struktur mit HTML
- Gestaltung mit CSS
- kleine Praxisbeispiele
- direkt sichtbare Ergebnisse

## 2) Einstiegsblock
### Ziel
Neue Nutzer klar in die HTML/CSS-Grundlagen führen.

### Empfohlene Kacheln
- HTML-Grundlagen
- HTML-Semantik und Struktur
- CSS-Grundlagen

### Inhaltlich passend zu bisherigen Planungen
- `html-grundlagen`
- `html-semantik-und-struktur`
- `css-grundlagen`

## 3) Große Themenblöcke
Die Web-Development-Landingpage sollte die Fachstruktur sichtbar in Hauptsektionen gliedern:

### A. HTML-Einstieg
- HTML-Grundlagen
- HTML-Semantik und Struktur
- Links, Bilder und Pfade

### B. CSS-Einstieg
- CSS-Grundlagen
- Box-Modell und Layouts

### C. Praxis und Anwendung
- Code-/Preview-Logik
- kleine Kombi-Aufgaben
- spätere Projekt- oder Komponentenblöcke

### D. Vertiefungen
- responsive Themen später
- größere Web-/UI-Praxis später

## 4) Wissensseiten-Bereich
### Ziel
Die wichtigsten `articles` des Fachbereichs sichtbar machen.

### Empfehlung
Keine Vollauflistung, sondern:
- 5–7 kuratierte Kernseiten
- sichtbar nach Einstieg / Praxis gegliedert

### Kartenelemente
- Titel
- Kurzbeschreibung
- Lernstufe
- Sektion

## 5) Aufgaben-Bereich
### Ziel
Web-Aufgaben sollen als praktische Lernform direkt sichtbar sein.

### Empfohlene Sektionen
- `HTML üben`
- `CSS verstehen`
- `Pfade und Struktur trainieren`
- `Kleine Code-Challenges`

### Inhalt pro Karte

- Kurzbeschreibung
- Schwierigkeit
- optional Typ: Code / Zuordnen / Preview / Challenge

### Ziel
Web Development soll nicht wie eine Referenzwand wirken.

- `Weiter üben und ausbauen`

#### Zum Einstieg
- HTML-Grundlagen
- HTML-Semantik
- CSS-Grundlagen

#### Weiter üben und ausbauen
- Box-Modell
- Layoutlogik
- Code-/Preview-Übungen
- spätere Mini-Projekte

## 7) Aufgaben-Hub-Verknüpfung
### Ziel
Web-Fachseite und globaler Aufgaben-Hub sollen sich gegenseitig verstärken.

### Empfehlung
Auf der Fachseite sichtbar:
- `Alle Web-Development-Aufgaben ansehen`
- Link zum globalen `/aufgaben`-Hub mit Fachfilter Web Development

## Was vermieden werden sollte
- Web Development nur als HTML/CSS-Liste ohne Praxiszugang
- spätere Projekte schon vor den Grundlagen priorisieren
- Aufgaben hinter Wissensseiten verstecken
- Fachbereich ohne direkte Code-/Preview-Logik darstellen

## Zusammenfassung
Die Fach-Landingpage `web-development` sollte Web Development als klaren Einstieg in strukturierte Web-Grundlagen zeigen:
- HTML verstehen
- CSS anwenden
- direkt ausprobieren
- später ausbauen

Damit wird aus HTML/CSS kein Altbestand mehr, sondern ein moderner, klar navigierbarer Web-Einstieg.
