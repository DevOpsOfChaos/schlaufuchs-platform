# Informatik – Zielseite `compiler-interpreter-und-programmierparadigmen`

## Zweck dieser Datei
Diese Datei beschreibt die konkrete Zielseite `informatik/compiler-interpreter-und-programmierparadigmen` für den Relaunch.

Sie ist **kein Altseiten-Import**, sondern eine neue Produktspezifikation für eine verständliche Wissensseite zu Compiler, Interpreter und grundlegenden Programmierparadigmen.

Die Seite soll diese beiden Alt-Themen in einer modernen, kompakten Form zusammenführen, weil sie im Einstieg eng zusammenhängen: Wie Programme ausgeführt werden und wie Programme grundsätzlich gedacht oder organisiert werden.

## Rolle der Seite im neuen Informatik-Aufbau
Die Seite gehört in den Abschnitt:
- Informatik
- Einstieg in Programmierung

Sie kommt sinnvoll **nach** `programmieren-mit-c-und-python` und **vor** den konkreten C-Grundlagenseiten.

## Lernziel der Seite
Nach der Seite sollen Lernende verstehen:
- was ein Compiler grob macht
- was ein Interpreter grob macht
- dass Programmiersprachen unterschiedlich in ausführbaren Code überführt werden
- was prozedurales und objektorientiertes Denken grob bedeutet
- warum diese Unterschiede im Einstieg relevant sind

## Zielcharakter der neuen Seite
Die Seite soll:
- zwei oft trocken erklärte Themen verständlich zusammenführen
- keine überladene Theorieseite werden
- mit klaren Vergleichen und wenigen starken Beispielen arbeiten
- mit einem verpflichtenden Merksatz enden

## Empfohlener Seitenaufbau
## 1) Hero / Einstieg
- Titel: `Compiler, Interpreter und Programmierparadigmen`
- kurzer Einleitungssatz
- 2–3 Lernziele

## 2) Abschnitt: Wie wird aus Quellcode ein lauffähiges Programm?
- Quellcode als Ausgangspunkt
- Übersetzung oder Ausführung
- Grundidee von Compiler und Interpreter

## 3) Abschnitt: Compiler einfach erklärt
- übersetzt Programmcode in ausführbare Form
- typische Einordnung bei C
- Fokus auf Grundprinzip, nicht Tooldetails

## 4) Abschnitt: Interpreter einfach erklärt
- führt Code schrittweise oder laufzeitnah aus
- typische Einordnung bei Python
- ebenfalls ohne Tiefentechnik

## 5) Abschnitt: Programmierparadigmen grob einordnen
- prozedural: schrittweiser Ablauf
- objektorientiert: Arbeit mit Objekten und Zuständen
- nur Grundidee, keine lange Theorie

## 6) Abschnitt: Warum das für den Einstieg hilft
- unterschiedliche Werkzeuge besser verstehen
- C und Python sinnvoll einordnen
- spätere Lernwege klarer sehen

## 7) Lernmodus / interaktive Module
- Compiler oder Interpreter zuordnen
- prozedural oder objektorientiert grob unterscheiden
- Ablaufgrafik in richtige Reihenfolge bringen

## 8) Merksatz
> Compiler und Interpreter beschreiben, wie Programmcode ausgeführt wird, während Programmierparadigmen beschreiben, wie Programme grundsätzlich aufgebaut oder gedacht werden.
